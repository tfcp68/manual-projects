Решение через ДП назад.

Рекуррентная формула: **$a_{i}= a_{i-1}+ a_{i-2}+ a_{i-3}+...+ a_{i-k}$**

```
Псевдокод:

//a - массив состоящий из количества способов добраться до каждого i-го столбика
//k - длина максимального прыжка
//n - столбик, до которого необходимо дойти

функция кузнечик_и_прыжок_длиной_k(n, k)
	для i от 1 до n
		s <- 0
		r <- минимум из k и i
		для j от 1 до r      
			s <- s + a[i - j]
		a[i] <- s 
    вернуть a[n]
```
```python
#python

def grig_and_k_step_back(n, k):
    if n < 0 or k < 0:
        return -1
    a = [1]
    for i in range(1, n + 1):
        s = 0
        r = min(k, i)
        for j in range(1, r + 1):
            s += a[i - j]
        a.append(s)
    return a[n]
```
```cpp
//C++
//sport version

int grig_and_k_step_back(int n, int k) {
    if (n < 0 or k < 0) {
        return -1;
    }
    std::vector<int> a(n + 1, 0);
    a[0] = 1;

    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= std::min(i, k); j++) {
            a[i] = a[i] + a[i - j];
        }
    }
    return a[n];
}
```

Решение через ДП вперёд:

Формулы для формирование последующих значений: $a_{i+1}=a_{i+1}+a_{i}$, $a_{i+2}=a_{i+2}+a_{i},...a_{i+k}=a_{i+k}+a_{i}$

```
Псевдокод:

//a - массив состоящий из количества способов добраться до каждого i-го столбика
//k - длина максимального прыжка
//n - столбик, до которого необходимо дойти

функция кузнечик и прыжок длиной k(n, k)
	a <- [1]
	a <- a + [0] * (n + k + 1)
	для i от 0 до n
		 для j от 1 до k
			 a[i + j] <- a[i + j] + a[i]
	вернуть a[n]	 
```
```python
#python

def grig_and_k_step_forward(n, k):
    if n < 0 or k < 0:
        return -1
    a = [1]
    a = a + [0] * (n + k + 1)
    for i in range(n+1):
        for j in range(1, k + 1):
            a[i + j] += a[i]
    return a[n]
```
